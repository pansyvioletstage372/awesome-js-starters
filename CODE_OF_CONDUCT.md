# Code of Conduct

## Our Pledge

This is a community-first project. Everyone who contributes — whether it's their first open source PR or their hundredth — deserves a respectful and constructive experience.

## Standards

**Encouraged behavior:**
- Sharing honest, real-world experience with packages (including limitations)
- Providing constructive feedback on submissions
- Helping newcomers format their first contribution
- Suggesting better alternatives if you know of them

**Not acceptable:**
- Dismissing a submission because you prefer a different package
- Self-promotion without genuine user perspective
- Spam, off-topic issues, or low-effort submissions
- Harassment of any kind

## Enforcement

Violations can be reported by opening a private issue or contacting the maintainer directly via GitHub. Maintainers reserve the right to remove content or ban contributors who repeatedly violate these standards.

## Scope

This Code of Conduct applies to all repository spaces: issues, pull requests, discussions, and any related community spaces.
